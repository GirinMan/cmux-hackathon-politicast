# 02. NVIDIA NeMo Data Designer 합성 파이프라인

- 작성일: 2026-04-26
- 데이터셋: `nvidia/Nemotron-Personas-Korea` v1.0
- 작성자: provenance-research

## 1. 한 줄 요약

Nemotron-Personas-Korea는 **NeMo Data Designer**(NVIDIA 오픈소스 합성 데이터 프레임워크)에서 **(a) 한국 공공 통계로 학습된 PGM 샘플러 + (b) OCEAN(Big-5) 성격 모형 샘플러 + (c) Gemma-4-31B-IT (Apache-2.0) 두 차례 호출**을 조합해 만든 dataset이다. README의 `images/nemotron_personas_korea_approach.png` 다이어그램이 이 파이프라인을 명시적으로 보여준다.

## 2. NeMo Data Designer 개요

| 항목 | 값 | 출처 |
|---|---|---|
| 이름 | NeMo Data Designer | README.md L96 [^1] |
| 저장소 | https://github.com/NVIDIA-NeMo/DataDesigner | README.md L96 [^1] |
| 라이선스 | Apache-2.0 (오픈소스 공개 2025-12) | Hacker News post by Ben Van Segbroeck [^2] |
| 출자 | Gretel(2025년 NVIDIA 인수)에서 시작 → Nemotron 데이터 생산을 위한 사내 프레임워크가 된 후 OSS화 | Hacker News [^2], YouTube talk [^3] |
| 기능 | 통계 샘플러(범주/Gaussian/Bernoulli/Poisson 등) + LLM 컬럼 + Person 샘플러 + 의존-aware 컬럼 + 검증 | NVIDIA NeMo Microservices Docs [^4][^5] |
| 출력 포맷 | CSV / JSON / Parquet | NVIDIA Docs [^4] |

### 2.1 핵심 빌딩 블록 — Sampler 종류 (NeMo Microservices 문서 [^4])

- 통계 샘플러: Category, Subcategory, Uniform, Gaussian, Bernoulli, Bernoulli Mixture, Binomial, Poisson, SciPy 분포
- 시간 샘플러: DateTime, Timedelta
- ID: UUID
- **Person 샘플러**: 인구통계 + 성격 + 컨텍스트 페르소나를 통합 생성 — Korea 판이 사용한 핵심 모듈

### 2.2 Person 샘플러의 두 가지 모드 (NVIDIA Docs [^5][^6])

- **PGM 모드 (en_US, ja_JP, hi_IN, en_IN, *그리고 Korea가 추가됨*):** Gretel/NVIDIA 자체 PGM(Probabilistic Generative Model)이 인구조사 데이터 분포를 학습 → 인구통계 일관성과 변수 간 상관관계 보존
- **Faker 모드 (그 외 locale):** 통계적 정확도 없이 이름/주소만 합성

> NVIDIA Docs ([^6]): *"The best quality person / synthetic persona data is generated by sampling a superset of the Nemotron-Personas datasets. This is currently supported for the following locales: 'en_US', 'ja_JP', 'hi_IN', and 'en_IN'."*
> Korea (`ko_KR`) 판은 본 README에 따르면 같은 PGM 패턴을 한국 공공 통계로 재학습한 것이다.

## 3. Korea 판 파이프라인 — 다이어그램 해석

`images/nemotron_personas_korea_approach.png` (한국어 라벨, NeMo Data Designer 점선 박스 안)을 좌→우, 상→하 흐름으로 옮기면:

### 3.1 단계별 역할

| 단계 | 입력 | 출력 컬럼 | 모듈 |
|---|---|---|---|
| **PGM 샘플링** | KOSIS / 대법원 / NHIS / KREI 의 분포 | `sex`, `age`, `marital_status`, `military_status`, `family_type`, `housing_type`, `education_level`, `bachelors_field`, `occupation`, `district`, `province`, `country` (12 demographic + geo) | NVIDIA/Gretel 자체 PGM (Apache-2.0 라이선스) |
| **OCEAN 샘플링** | Big-5 인구분포 통계 (출처 미공개, NVIDIA 내부 추정) | (직접 컬럼은 아니고 LLM A에 prompt로 주입되는 latent 성격 점수) | OCEAN(=Big-5) 통계 모형 — Openness, Conscientiousness, Extraversion, Agreeableness, Neuroticism [^7][^8] |
| **LLM A 호출** | PGM 결과 + OCEAN 결과 + 문화적 컨텍스트 | `cultural_background`, `skills_and_expertise`(+`_list`), `career_goals_and_ambitions`, `hobbies_and_interests`(+`_list`) — 6개 attribute 필드 | `google/gemma-4-31B-it` (Apache 2.0) [^1] |
| **LLM B 호출** | 1차 attribute + 인구통계 컨텍스트 | `persona`(요약), `professional_persona`, `sports_persona`, `arts_persona`, `travel_persona`, `culinary_persona`, `family_persona` — 7 페르소나 텍스트 | `google/gemma-4-31B-it` |
| **검증** | 모든 컬럼 | 분포/스키마/제약 위반 검증, 필요 시 재샘플 | Data Designer 내장 validator [^4] |

→ 결과적으로 `1 record × 7 persona narrative = 7M personas` 산출.

## 4. 핵심 LLM: `google/gemma-4-31B-it`

| 항목 | 값 | 출처 |
|---|---|---|
| 모델 | Gemma 4 31B Instruction-Tuned | README.md L106 [^1] |
| 출시 | 2026-04-02 | Implicator [^9], Maarten Grootendorst [^10] |
| 라이선스 | **Apache 2.0** (Gemma 시리즈 최초의 표준 OSS 라이선스) | Implicator [^9] |
| 아키텍처 | 31B dense, 60-layer, 5:1 local/global attention 인터리빙, K=V 글로벌, p-RoPE, GQA(8 Q : 1 KV) | Grootendorst [^10] |
| 컨텍스트 | 256K 토큰 (Vertex AI 카드), 1024 sliding window | Vertex AI Docs, Grootendorst [^10] |
| 다국어 | Gemma 3에서 140+ 언어 지원, Korea 한국어 narrative 품질 충분 | Google AI for Developers [^11] |
| 입력 | Text + Image (Korea 판은 text만 사용) | Vertex AI [^12] |

**중요한 라이선스 함의:** Gemma 4가 Apache-2.0으로 라이선스됨에 따라 NVIDIA는 합성 데이터에 대한 라이선스 격오/seed 클로즈가 없어졌고 (이전 Gemma 라이선스의 'derivative trained on Gemma output' 조항 제거), Korea 판을 CC BY 4.0으로 자유 배포할 수 있게 되었다 [^9]. PolitiKAST는 이를 페르소나 추출/재가공 → 다른 모델 fine-tune에 자유롭게 활용 가능 (CC BY 4.0의 attribution 의무만 만족하면 됨).

## 6. 검증/품질 통제 단계 (Data Designer 내장 [^4])

- **Schema Validation**: 컬럼 타입/제약 검사
- **Relationship Consistency**: 의존성 컬럼 간 논리 일관성 (예: `bachelors_field`가 비어있으면 `education_level`이 학사 미만이어야)
- **Distribution Fidelity**: 원 통계 분포와의 유사도 검증

PolitiKAST 활용 시 함의: 이미 PGM 단계에서 `(province, age, sex)` 결합 분포가 KOSIS와 정합하므로 5 contract region(서울시장/광주시장/대구시장/부산 북구 갑/대구 달서구 갑) 표본 추출은 그대로 사용 가능. 단, **변수 간 독립성 가정**이 명시되어 있어(README L154) `(sex × bachelors_field)` 결합 분포 등은 실제 한국과 다를 수 있음 — 정치성향 추정 시 주의.

## 7. 한계 (README + Docs 종합)

| 한계 | 영향 | PolitiKAST 대응 |
|---|---|---|
| 변수 간 독립성 가정 (예: 성별×전공 결합 효과 무시) | 교차분석 시 편향 | 가능한 한 marginal 분포만 사용 |
| 19세 이상만 포함 (한국 법정 성년) | 18세 유권자 누락 | 19+ 정책 명시 + Limitations에 기재 |
| 생물학적 sex만 있고 gender 미수록 (한국 공공통계 부재) | 성별 정체성/소수자 표현 불가 | Limitations 명시 |
| 시의성: KOSIS/대법원 데이터 시점 미명시 (README는 2024 KREI/2024 NHIS만 명시) | 2026년 정치적 맥락과 시차 가능 | KG에 timestamp 메타데이터 부여 |
| 엔터프라이즈(금융/헬스케어) 페르소나 제외 | 직업 분포 일부 결측 | KOSIS 직업 분포로 보완 |

## 8. 미확인 항목

- PGM의 정확한 구조(베이지안 네트워크 vs HMM vs Markov Random Field 등) — NVIDIA OSS 코드를 확인하면 추정 가능, 본 노트에서는 미수행
- LLM A/B의 정확한 prompt 템플릿 — README에 미공개, Data Designer 코드 분석 필요 (해커톤 시간 외)
- 검증 단계에서 LLM-as-judge 사용 여부 — Hacker News [^2]는 "optional LLM-as-judge scoring" 가능하다고 언급하나 Korea 판에 적용했는지는 미공개
